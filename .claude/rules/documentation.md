ドキュメント作成時は下記を遵守してください。

- 日本語で書いてください。
- 図は mermaid で書いてください。
- 技術文書としてふさわしいフォーマル寄りの文体で文章を執筆してください。
- 専門用語が出現したときは適宜その意味を補完するよう努めてください。
- 半角の文字列と、全角の文字列の間にはスペースを挿入してください。
  - ただし句読点の直後であればスペース挿入は省略して良いものとします。検索性を高めるためです。
- 全角かっこ、全角コロンは決して使用しないでください。
  - 半角かっこを使用する際、前かっこでは `[space](` の形で、後ろかっこでは `)[space]` の形でスペースを挿入してください。半角コロンを使用する際は `:[space]` の形でスペースを挿入してください。

```
悪い例: これをARGといいます。
良い例1: これを ARG といいます。
良い例2: これを、ARG といいます。
```

```
悪い例1: 代替現実ゲーム（ARG）は、
悪い例2: 代替現実ゲーム ( ARG ) は、
良い例: 代替現実ゲーム (ARG) は、
```

```
悪い例1: ARG：代替現実ゲーム
悪い例2: ARG : 代替現実ゲーム
良い例: ARG: 代替現実ゲーム
```

- ダブルクオーテーションは日本語文章中には決して使用しないでください。
- 「ひとつひとつ」「一発逆転」のような日本語の言葉を、「 1 つ 1 つ」「1発逆転」のように算用数字で書いてはいけません。
  - 「第 1 回」「第 2 回」、「1 点のメリット」「2 点のメリット」のように、異なる数に置き換えられるものは算用数字で書いてかまいません。
  - 1,000 のように、 4 桁以上の数にはコンマを付けてください。
- 衒学的な表現はなるべく避け、平易な日本語で記載してください。
  - 「〇〇を実現します」 「〇〇を実施します」などの表現は、「〇〇できます」 「〇〇します」 という表現で書けないか常に検討してください。冗長な表現を避け端的に表現してください。

```
悪い例: 解説が行われました
```

```
良い例: 解説されました
```

- 「〇〇で、〇〇」のように並列的に並べることで冗長な表現になることを必ず避けてください。
  - 一文の長さが長くなりすぎないよう適切に区切ってください。

```
悪い例: Python はインタプリタ言語で、低速です。
```

```
良い例: Python はインタプリタ言語です。また、低速です。
```

- むやみに箇条書きを挿入すべきではありません。日本語文の途中で箇条書きが挿入されるのは日本語文章として不自然だからです。
  - 文の形で完結させるよう努め、項目の並列性がある場合のみ箇条書きを使用してください。

悪い例:
```
検証環境として
* Windows
* Next.js
を使用しました。
```

良い例1:
```
検証環境として、Windows と Next.js を使用しました。
```

良い例2:
```
検証環境は以下の通りです。
* Windows
* Next.js
```

- 助詞を適切に使用してください。特に、「以下」「次」「は」のワードについては細心の注意を払ってください。

悪い例
```
設定内容は次とします。
```

良い例
```
設定内容は次のようにします。
```

悪い例
```
get() は、ユーザーが情報を取得します。
```

良い例 (ユーザーを主語にする場合)
```
get() では、ユーザーが情報を取得します。
```

良い例 (get を主語にする場合)
```
get() は、ユーザーが情報を取得するための関数です。
```

- 「整理する」「包括的に」「一気通貫で」「ポイントは」「要点は」「寄せる」といった「AI 感のある」言葉、あるいは「直訳感のある」言葉の使用を避けてください。
  - 日本語文章において一般的に使用されているワーディングを意識してください。

- 抽象度を適切にコントロールしてください。読み手の認知負荷を下げるための工夫です。読み手を裏切ってはいけません。
  - まずは抽象度の高い書き方で全体を説明しながら、カスケード的に、より抽象度の低い話へと話を進めていくよう、読み手の認知の順番を意識してください。
  - たとえば「序論→本論→結論」 「背景→提案→検証→考察」のように、一般的な技術文書テンプレートにしたがって文章を構成してください。

- 手順を説明するときはいきなり手順の中身を説明する前に、簡単にでもよいので手順を踏む理由や目的を説明してください。
- 見出しには「環境変数の設定」など、内容を実態に即した具体的な見出し名をつけてください。
- 同じものは同じ言葉を使って説明してください。言い換えてはいけません。
